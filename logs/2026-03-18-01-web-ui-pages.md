# 2026-03-18-01: 웹 UI 2페이지 구현

## 요청
페르소나 관리 페이지와 피드백 생성 페이지를 별도 웹 페이지로 구성.

## 수행 내용

### 라우팅
- `react-router-dom` 도입
- `/` → 피드백 페이지, `/personas` → 페르소나 관리 페이지
- `Layout` 컴포넌트: 공유 헤더 + NavLink 네비게이션

### 피드백 페이지 (`FeedbackPage`)
- 기존 `App.tsx` 내용을 `src/ui/pages/FeedbackPage.tsx`로 이동
- 게시글 입력 → 페르소나 선택 → 피드백 생성 (SSE 스트리밍)
- 이미지 미리보기, 에러/재시도 UI 유지

### 페르소나 페이지 (`PersonaPage`)
- **목록 모드**: `PersonaList` — 카드 그리드, 수정/삭제 버튼
- **추가/수정 모드**: `PersonaForm` — 이름, 톤, 감정, 격식, 프롬프트 힌트, 예시 패턴
- **삭제 확인**: 오버레이 모달

### 서버 CRUD API
- `POST /api/personas` — 새 페르소나 생성 (ID 자동 생성)
- `PUT /api/personas/:id` — 페르소나 수정
- `DELETE /api/personas/:id` — 페르소나 삭제
- 변경 시 인메모리 레지스트리 + JSON 파일 동시 업데이트

### 도메인 변경
- `personaRegistry.ts`: `addPersona`, `updatePersona`, `removePersona` 함수 추가
- `savePersonas.ts`: 카테고리별 JSON 파일 영속화

### 새 파일
- `src/ui/pages/Layout.tsx`, `FeedbackPage.tsx`, `PersonaPage.tsx`
- `src/ui/components/PersonaForm/`, `PersonaList/`
- `src/ui/hooks/usePersonaCrud.ts`
