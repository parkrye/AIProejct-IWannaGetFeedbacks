# 2026-03-18-14: 피드백 UX 4건 개선

## 요청
1. 실시간 스트리밍 제거 → 완료 후 일괄 표시. 대기 진척도 표시. 생성/오류 구분.
2. 그룹 선택 시 해당 그룹 페르소나만 표시. 전체 선택/해제 제거.
3. 헛소리 레벨이 높아도 게시글 내용과 관련된 헛소리를 하도록.
4. 동적 선택에서 매칭 점수 0일 때 레지스트리 순서 대신 랜덤 셔플.

## 수행 내용

### 1. 피드백 일괄 표시
- **서버**: SSE 이벤트 타입 변경 — `progress`(대기중), `feedback`(완료), `error`(실패) 3종
  - `progress`: `{ current, total, status, personaName }` — 현재 생성 중인 페르소나명 표시
  - `feedback`: 전체 content 한번에 전송 (토큰 스트리밍 제거)
  - `error`: 페르소나별 에러 메시지
- **프론트**: `useGeneration`에 `errors` Map 추가 (글로벌 에러 + 페르소나별 에러 분리)
- **FeedbackDisplay**: 로딩 스피너 + 프로그래스 바 → 완료된 카드만 표시 + 에러 카드 분리
- **FeedbackCard**: `isStreaming` prop 제거 (더 이상 스트리밍 없음)

### 2. 그룹 모드 개선
- **PersonaGroupSelector**: 그룹 클릭 시 `selectedGroupId` 상태 관리
  - 그룹 모드: 선택된 그룹의 멤버만 표시, 전체 선택/해제 버튼 숨김
  - 직접 선택 모드: 기존 동작 유지 (전체 표시 + 전체 선택/해제)
  - 그룹 미선택 시 "그룹을 선택하세요" 안내 표시

### 3. 헛소리 프롬프트 수정
- 변경 전: "게시글 내용과 상관없이 자유롭게 연상되는 내용을 작성하세요"
- 변경 후: "게시글의 주제나 키워드는 반드시 언급하되, 엉뚱하고 예상치 못한 방향으로 해석하여 댓글을 작성하세요"
- 중간 레벨도 "게시글 내용을 기반으로 하되" 명시

### 4. 동적 선택 랜덤화
- 매칭 점수 > 0인 페르소나가 maxCount 이상: 상위 n명
- 매칭 점수 > 0인 페르소나가 부족: 매칭 + 나머지에서 랜덤 보충
- 매칭 페르소나 0명: 전체에서 완전 랜덤 셔플
