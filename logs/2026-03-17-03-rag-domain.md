# 2026-03-17-03: RAG 도메인 구현

## 요청
라벨링된 SNS 대화 데이터(189,717건)를 활용한 RAG 파이프라인 구현.

## 수행 내용

### RAG 도메인 (`src/domains/rag/`)
- **types.ts**: NormalizedDialogue, VectorEntry, SearchResult, SearchFilter, RawDataAdapter
- **adapters/snsDialogueAdapter.ts**: SNS 대화 데이터 정규화 어댑터
- **embedder.ts**: @huggingface/transformers + multilingual-e5-small 래퍼
- **vectorStore.ts**: 인메모리 벡터 저장소 + 바이너리 파일 영속화
- **search.ts**: 2단계 검색 (메타데이터 필터 → 코사인 유사도)

### 빌드 스크립트
- `scripts/build-vectors.ts`: 데이터셋 → 임베딩 → 바이너리 파일 생성 (1회성)
- 진행률 표시, 속도/잔여시간 계산, 에러 핸들링

### 서버 통합
- `server/app.ts`: 서버 시작 시 벡터 스토어 자동 로딩
- `generateRoute.ts`: RAG 검색 결과를 프롬프트에 주입
- `promptBuilder.ts`: `[실제 SNS 대화 참고]` 섹션 추가

### 테스트
- snsDialogueAdapter: 정규화 검증 (2개)
- vectorStore: 추가/저장/로딩/차원 검증 (3개)
- search: 유사도 순위, 메타필터, 폴백 (4개)

## 테스트 결과
- **13개 파일, 50개 테스트 전체 통과**
- TypeScript 컴파일 에러 없음

## 사용법
```bash
# 1회성 임베딩 생성 (30분~1시간)
npm run build-vectors -- "C:\path\to\VL"

# 서버 시작 (벡터 스토어 자동 로딩)
npm run server
```
